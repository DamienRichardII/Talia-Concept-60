# TALIA Concept — Validation du site actuel avant ajout de la page Catalogue

## Objet
Ce fichier valide l'état actuel du site TALIA Concept comme base approuvée avant l'ajout d'une nouvelle page `catalogue.html`.

Toute modification future doit respecter ce document afin d'éviter toute auto-régression.

---

## Base validée

Le site actuel est validé sur les points suivants :

- direction artistique premium globale
- palette actuelle
- typographies actuelles
- hiérarchie visuelle générale
- structure de la home page
- section hero actuelle dans son intention générale
- section "Notre histoire" dans son intention générale
- section "Nos univers"
- galerie de réalisations
- logique de vente directe
- responsive existant
- ton éditorial global
- CTA existants
- cohérence premium générale

---

## Objectif de la prochaine mission

La prochaine mission a uniquement pour but :

1. d'ajouter une nouvelle page dédiée au **catalogue fournisseur**
2. de relier proprement `index.html` à cette nouvelle page
3. d'ajouter les liens de navigation nécessaires
4. de conserver intacte l'expérience actuelle du site

---

## Modifications autorisées

Sont autorisés uniquement :

- création d'un nouveau fichier `catalogue.html`
- ajout d'un lien **Catalogue** dans le header
- ajout d'un lien **Catalogue** dans le menu mobile
- ajout d'un lien **Catalogue fournisseur** dans le footer
- liaison entre `index.html` et `catalogue.html`
- liaison entre les univers de la home et les sections pertinentes du catalogue
- ajustements techniques minimes nécessaires à cette intégration
- éventuelles ancres propres sur la page catalogue
- éventuels CTA ciblés liés au catalogue sur la nouvelle page

---

## Modifications interdites

Il est interdit de :

- refaire la home page
- changer la direction artistique du site
- changer la palette
- changer les typographies
- casser la galerie existante
- casser les sections existantes
- déplacer lourdement les blocs validés
- modifier profondément le hero
- modifier profondément "Notre histoire"
- casser la logique de vente directe
- réintroduire toute logique showroom
- casser le responsive
- remplacer les CTA existants
- créer une page catalogue cheap, brute ou incohérente avec le reste du site
- transformer la page catalogue en tableau administratif pauvre
- ajouter des sections inutiles sur la home
- introduire des liens morts ou ancres cassées

---

## Exigences pour la nouvelle page Catalogue

La future page catalogue doit :

- être premium
- être cohérente avec le site existant
- reprendre les codes visuels principaux du site
- être responsive
- être lisible
- valoriser la richesse du catalogue fournisseur
- être structurée par univers et sous-catégories
- rassurer sur la qualité des partenaires et marques
- intégrer des CTA propres vers la prise de contact ou le devis

---

## Exigences de navigation

Après ajout du catalogue, le site devra comporter :

### Header
- un lien ou CTA **Catalogue**

### Menu mobile
- un lien **Catalogue**

### Footer
- un lien **Catalogue fournisseur**

### Index → Catalogue
- `index.html` doit être relié proprement à `catalogue.html`

### Univers → Catalogue
- les univers mis en avant sur la home doivent pointer vers la page catalogue
- idéalement vers les sections correspondantes via ancres propres

---

## Validation anti-régression

Avant toute livraison, vérifier obligatoirement :

### A. Home page
- [ ] aucune régression visuelle sur la home
- [ ] aucun bloc validé cassé
- [ ] aucune section déséquilibrée
- [ ] aucun CTA existant dégradé

### B. Navigation
- [ ] lien Catalogue présent dans le header
- [ ] lien Catalogue présent dans le menu mobile
- [ ] lien Catalogue fournisseur présent dans le footer
- [ ] aucun lien mort

### C. Nouvelle page
- [ ] page `catalogue.html` créée
- [ ] cohérence visuelle avec le reste du site
- [ ] structure premium
- [ ] responsive complet
- [ ] contenu bien hiérarchisé
- [ ] CTA présents et cohérents

### D. Liaisons avec les univers
- [ ] les univers de la home renvoient bien au catalogue
- [ ] les ancres éventuelles sont propres et fonctionnelles
- [ ] aucun comportement cassé sur desktop
- [ ] aucun comportement cassé sur mobile

### E. Cohérence métier
- [ ] aucune réintroduction du showroom
- [ ] site toujours centré sur vente directe / accompagnement / devis
- [ ] ton premium et professionnel conservé

---

## Condition de validation finale

Si un seul point ci-dessus n'est pas respecté, la mission est considérée comme incomplète.

Le site actuel doit rester la base approuvée.
La nouvelle page catalogue doit s'ajouter proprement au système existant, sans casser l'existant.
